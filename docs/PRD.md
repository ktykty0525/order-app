# 커피 주문 앱

## 1. 프로젝트 개요

### 1.1 프로젝트 명
커피 주문 앱

### 1.2 프로젝트 목적
사용자가 커피 메뉴를 주문하고, 관리자가 주문을 관리할 수 있는 간단한 풀스택 웹 앱

### 1.3 개발 범위
- 주문하기 화면(메뉴 선택 및 장바구니 기능)
- 관리자 화면(재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능

## 2. 기술 스택
- 프런트 엔드 : HTML, CSS, 리액트, 자바스크립트
- 백엔드 : Node.js, Express
- 데이터베이스 : PostgreSQL

## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음

## 4. 주문하기 화면 PRD

### 4.1 화면 개요
사용자가 커피 메뉴를 선택하고 장바구니에 담아 주문할 수 있는 메인 화면입니다.

### 4.2 화면 레이아웃

#### 4.2.1 헤더 영역
- **위치**: 화면 상단
- **구성 요소**:
  - 좌측: "COZY" 브랜드 로고 (다크 그린 배경의 사각형 박스)
  - 우측: 두 개의 버튼
    - "주문하기" 버튼 (라이트 그레이 배경, 다크 그린 테두리)
    - "관리자" 버튼 (라이트 그레이 배경, 다크 그린 테두리)

#### 4.2.2 메뉴 아이템 영역
- **위치**: 헤더 아래, 화면 중앙
- **레이아웃**: 메뉴 카드들이 가로로 배열됨
- **카드 구성 요소**:
  - 상단: 메뉴 이미지 영역 (플레이스홀더)
  - 제품명: 굵은 글씨로 표시
  - 가격: 한국 원화 형식으로 표시 (예: "4,000원")
  - 설명: "간단한 설명..." 플레이스홀더
  - 옵션 체크박스:
    - "샷 추가 (+500원)" - 선택 시 추가 요금 표시
    - "시럽 추가 (+0원)" - 추가 요금 없음
  - 하단: "담기" 버튼 (라이트 그레이 배경)

#### 4.2.3 장바구니 영역
- **위치**: 화면 하단
- **구성 요소**:
  - 제목: "장바구니" (좌측 상단)
  - 장바구니 아이템 목록:
    - 각 아이템은 제품명, 옵션 정보, 수량, 가격을 표시
    - 형식: "제품명 (옵션) X 수량" + 가격
    - 예시: "아메리카노(ICE) (샷 추가) X 1" - "4,500원"
  - 총 금액: "총 금액" 레이블과 함께 굵은 글씨로 표시
  - "주문하기" 버튼: 라이트 그레이 배경, 다크 그레이 테두리

### 4.3 기능 요구사항

#### 4.3.1 메뉴 표시
- 서버에서 메뉴 목록을 조회하여 화면에 표시
- 각 메뉴는 카드 형태로 표시
- 메뉴 이미지, 이름, 가격, 설명 정보 표시

#### 4.3.2 옵션 선택
- 각 메뉴 카드에서 옵션을 체크박스로 선택 가능
- "샷 추가" 옵션 선택 시 추가 요금(+500원)이 적용됨
- "시럽 추가" 옵션은 추가 요금 없음
- 여러 옵션을 동시에 선택 가능

#### 4.3.3 장바구니 기능
- "담기" 버튼 클릭 시 선택한 메뉴와 옵션이 장바구니에 추가됨
- 같은 메뉴와 옵션 조합이 이미 장바구니에 있으면 수량이 증가
- 장바구니에 담긴 아이템 목록을 실시간으로 표시
- 각 아이템의 총 가격 계산 및 표시
  - 기본 가격 + 옵션 추가 요금 × 수량
- 장바구니의 총 금액을 자동으로 계산하여 표시

#### 4.3.4 주문하기
- 장바구니 하단의 "주문하기" 버튼 클릭 시 주문 처리
- 주문 정보를 서버로 전송
- 주문 완료 후 적절한 피드백 제공 (성공 메시지 등)

### 4.4 UI/UX 요구사항

#### 4.4.1 색상
- 브랜드 컬러: 다크 그린 (헤더 로고, 버튼 테두리)
- 배경: 라이트 그레이 (버튼, 장바구니 영역)
- 텍스트: 기본 검정색, 총 금액은 굵은 글씨

#### 4.4.2 레이아웃
- 반응형 디자인 고려 (모바일/데스크톱)
- 메뉴 카드는 가로 스크롤 또는 그리드 레이아웃으로 배치
- 장바구니는 화면 하단에 고정 또는 스크롤 가능한 영역

#### 4.4.3 인터랙션
- 버튼 클릭 시 시각적 피드백 제공 (호버 효과 등)
- 장바구니에 아이템 추가 시 즉시 반영
- 총 금액은 실시간으로 업데이트

### 4.5 데이터 구조

#### 4.5.1 메뉴 데이터
```javascript
{
  id: number,
  name: string,
  price: number,
  description: string,
  imageUrl: string
}
```

#### 4.5.2 장바구니 아이템 데이터
```javascript
{
  menuId: number,
  menuName: string,
  basePrice: number,
  options: {
    addShot: boolean,  // +500원
    addSyrup: boolean  // +0원
  },
  quantity: number,
  totalPrice: number
}
```

#### 4.5.3 주문 데이터
```javascript
{
  items: [
    {
      menuId: number,
      menuName: string,
      options: {
        addShot: boolean,
        addSyrup: boolean
      },
      quantity: number,
      price: number
    }
  ],
  totalAmount: number,
  orderDate: timestamp
}
```

### 4.6 사용자 시나리오

1. 사용자가 화면에 접속하면 메뉴 목록이 표시됨
2. 원하는 메뉴 카드에서 옵션을 선택 (선택사항)
3. "담기" 버튼을 클릭하여 장바구니에 추가
4. 장바구니에 아이템이 추가되고 총 금액이 업데이트됨
5. 추가로 다른 메뉴를 선택하여 장바구니에 담을 수 있음
6. 장바구니 하단의 "주문하기" 버튼을 클릭하여 주문 완료
7. 주문 완료 후 성공 메시지 표시 (또는 주문 내역 화면으로 이동)

## 5. 관리자 화면 PRD

### 5.1 화면 개요
관리자가 주문을 관리하고 재고를 조회 및 수정할 수 있는 관리자 전용 화면입니다.

### 5.2 화면 레이아웃

#### 5.2.1 헤더 영역
- **위치**: 화면 상단
- **구성 요소**:
  - 좌측: "COZY" 브랜드 로고 (다크 그린 배경의 사각형 박스)
  - 우측: 두 개의 버튼
    - "주문하기" 버튼 (라이트 그레이 배경, 다크 그린 테두리)
    - "관리자" 버튼 (라이트 그레이 배경, 다크 그린 테두리, 현재 선택된 상태로 표시)

#### 5.2.2 관리자 대시보드 영역
- **위치**: 헤더 바로 아래
- **제목**: "관리자 대시보드"
- **구성 요소**: 주문 상태별 통계 정보
  - 형식: "총 주문 N / 주문 접수 N / 제조 중 N / 제조 완료 N"
  - 각 상태별 주문 건수를 실시간으로 표시
  - 예시: "총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0"

#### 5.2.3 재고 현황 영역
- **위치**: 관리자 대시보드 아래
- **제목**: "재고 현황"
- **레이아웃**: 메뉴별 재고 카드들이 가로로 배열됨
- **카드 구성 요소**:
  - 메뉴명: 각 메뉴의 이름 (예: "아메리카노 (ICE)", "아메리카노 (HOT)", "카페라떼")
  - 현재 재고 수량: "N개" 형식으로 표시 (예: "10개")
  - 재고 조정 버튼:
    - "+" 버튼: 재고 증가
    - "-" 버튼: 재고 감소
    - 버튼은 재고 수량 아래에 배치

#### 5.2.4 주문 현황 영역
- **위치**: 화면 하단
- **제목**: "주문 현황"
- **구성 요소**: 주문 목록
  - 각 주문 항목은 다음 정보를 포함:
    - 주문 일시: "월 일 시:분" 형식 (예: "7월 31일 13:00")
    - 주문 내역: "메뉴명(옵션) x 수량" 형식 (예: "아메리카노(ICE) x 1")
    - 주문 금액: "N,000원" 형식으로 표시 (예: "4,000원")
    - 액션 버튼: "주문 접수" 버튼 (주문 상태에 따라 버튼 텍스트 변경 가능)

### 5.3 기능 요구사항

#### 5.3.1 주문 통계 표시
- 서버에서 주문 데이터를 조회하여 통계 계산
- 주문 상태별로 분류하여 표시:
  - 총 주문: 모든 주문 건수
  - 주문 접수: 접수된 주문 건수
  - 제조 중: 현재 제조 중인 주문 건수
  - 제조 완료: 제조가 완료된 주문 건수
- 통계는 실시간으로 업데이트 (또는 주기적으로 갱신)

#### 5.3.2 재고 관리
- 각 메뉴별 현재 재고 수량을 서버에서 조회하여 표시
- "+" 버튼 클릭 시 해당 메뉴의 재고가 1 증가
- "-" 버튼 클릭 시 해당 메뉴의 재고가 1 감소
- 재고 변경 시 즉시 서버에 반영
- 재고 수량은 실시간으로 업데이트
- 재고가 0 이하로 내려가지 않도록 유효성 검사 (선택사항)

#### 5.3.3 주문 목록 표시
- 서버에서 주문 목록을 조회하여 표시
- 주문은 최신순으로 정렬하여 표시
- 각 주문의 상세 정보 표시:
  - 주문 일시
  - 주문한 메뉴와 옵션, 수량
  - 주문 금액
- 주문 상태에 따라 다른 액션 버튼 표시

#### 5.3.4 주문 상태 관리
- "주문 접수" 버튼 클릭 시 해당 주문의 상태를 "주문 접수"로 변경
- 주문 상태 변경 시 서버에 반영
- 주문 상태 변경 후 통계 정보 자동 업데이트
- 주문 상태별 다음 단계 버튼 제공 (예: "제조 시작", "제조 완료" 등)

### 5.4 UI/UX 요구사항

#### 5.4.1 색상
- 브랜드 컬러: 다크 그린 (헤더 로고, 버튼 테두리)
- 배경: 라이트 그레이 (버튼, 카드 영역)
- 텍스트: 기본 검정색
- 현재 선택된 탭(관리자)은 시각적으로 구분

#### 5.4.2 레이아웃
- 반응형 디자인 고려 (모바일/데스크톱)
- 재고 카드는 가로 스크롤 또는 그리드 레이아웃으로 배치
- 주문 목록은 세로로 스크롤 가능한 리스트 형태
- 각 영역은 명확하게 구분되어 표시

#### 5.4.3 인터랙션
- 버튼 클릭 시 시각적 피드백 제공 (호버 효과 등)
- 재고 변경 시 즉시 반영
- 주문 상태 변경 시 즉시 반영
- 통계 정보는 실시간으로 업데이트
- 주문 목록은 새 주문이 들어오면 자동으로 갱신 (또는 새로고침 버튼 제공)

### 5.6 사용자 시나리오

#### 5.6.1 재고 관리 시나리오
1. 관리자가 관리자 화면에 접속
2. 재고 현황 영역에서 각 메뉴의 현재 재고 수량 확인
3. 특정 메뉴의 재고를 조정하기 위해 "+" 또는 "-" 버튼 클릭
4. 재고 수량이 즉시 업데이트되고 서버에 반영
5. 변경된 재고 수량이 화면에 표시

#### 5.6.2 주문 관리 시나리오
1. 관리자가 관리자 화면에 접속
2. 관리자 대시보드에서 현재 주문 통계 확인
3. 주문 현황 영역에서 대기 중인 주문 목록 확인
4. 특정 주문의 "주문 접수" 버튼 클릭
5. 주문 상태가 "주문 접수"로 변경되고 통계가 업데이트됨
6. 주문 상태에 따라 다음 액션 버튼 표시 (예: "제조 시작", "제조 완료")
7. 주문 처리 완료 시 주문 상태를 "제조 완료"로 변경

#### 5.6.3 통계 모니터링 시나리오
1. 관리자가 관리자 화면에 접속
2. 관리자 대시보드에서 실시간 주문 통계 확인
3. 새로운 주문이 들어오면 통계가 자동으로 업데이트
4. 각 상태별 주문 건수를 모니터링하여 업무 효율성 파악

## 6. 백엔드 개발 PRD

### 6.1 데이터 모델

#### 6.1.1 Menus (메뉴)
커피 메뉴 정보를 저장하는 테이블입니다.

**필드:**
- `id` (INTEGER, PRIMARY KEY, AUTO_INCREMENT): 메뉴 고유 ID
- `name` (VARCHAR): 커피 이름 (예: "아메리카노(ICE)")
- `description` (TEXT): 메뉴 설명
- `price` (INTEGER): 기본 가격 (원 단위)
- `image_url` (VARCHAR): 이미지 URL
- `stock` (INTEGER): 재고 수량 (기본값: 0)
- `created_at` (TIMESTAMP): 생성 일시
- `updated_at` (TIMESTAMP): 수정 일시

**데이터 예시:**
```javascript
{
  id: 1,
  name: "아메리카노(ICE)",
  description: "시원한 아이스 아메리카노",
  price: 4000,
  image_url: "https://images.unsplash.com/...",
  stock: 10,
  created_at: "2024-01-01T00:00:00Z",
  updated_at: "2024-01-01T00:00:00Z"
}
```

#### 6.1.2 Options (옵션)
메뉴에 추가할 수 있는 옵션 정보를 저장하는 테이블입니다.

**필드:**
- `id` (INTEGER, PRIMARY KEY, AUTO_INCREMENT): 옵션 고유 ID
- `name` (VARCHAR): 옵션 이름 (예: "샷 추가", "시럽 추가")
- `price` (INTEGER): 옵션 추가 가격 (원 단위, 기본값: 0)
- `menu_id` (INTEGER, FOREIGN KEY): 연결된 메뉴 ID (NULL이면 모든 메뉴에 적용 가능)
- `created_at` (TIMESTAMP): 생성 일시
- `updated_at` (TIMESTAMP): 수정 일시

**데이터 예시:**
```javascript
{
  id: 1,
  name: "샷 추가",
  price: 500,
  menu_id: null,  // null이면 모든 메뉴에 적용 가능
  created_at: "2024-01-01T00:00:00Z",
  updated_at: "2024-01-01T00:00:00Z"
}
```

#### 6.1.3 Orders (주문)
주문 정보를 저장하는 테이블입니다.

**필드:**
- `id` (INTEGER, PRIMARY KEY, AUTO_INCREMENT): 주문 고유 ID
- `order_date` (TIMESTAMP): 주문 일시
- `total_amount` (INTEGER): 총 주문 금액 (원 단위)
- `status` (VARCHAR): 주문 상태 ('received', 'in_progress', 'completed')
- `created_at` (TIMESTAMP): 생성 일시
- `updated_at` (TIMESTAMP): 수정 일시

**데이터 예시:**
```javascript
{
  id: 1,
  order_date: "2024-12-27T14:30:00Z",
  total_amount: 12500,
  status: "received",
  created_at: "2024-12-27T14:30:00Z",
  updated_at: "2024-12-27T14:30:00Z"
}
```

#### 6.1.4 OrderItems (주문 상세)
주문에 포함된 메뉴와 옵션 정보를 저장하는 테이블입니다.

**필드:**
- `id` (INTEGER, PRIMARY KEY, AUTO_INCREMENT): 주문 상세 고유 ID
- `order_id` (INTEGER, FOREIGN KEY): 주문 ID
- `menu_id` (INTEGER, FOREIGN KEY): 메뉴 ID
- `menu_name` (VARCHAR): 메뉴 이름 (주문 시점의 이름 저장)
- `quantity` (INTEGER): 수량
- `unit_price` (INTEGER): 단가 (메뉴 가격 + 옵션 가격)
- `options` (JSON): 선택한 옵션 정보 (예: {"addShot": true, "addSyrup": false})
- `created_at` (TIMESTAMP): 생성 일시

**데이터 예시:**
```javascript
{
  id: 1,
  order_id: 1,
  menu_id: 1,
  menu_name: "아메리카노(ICE)",
  quantity: 2,
  unit_price: 4500,  // 4000 (기본) + 500 (샷 추가)
  options: {
    addShot: true,
    addSyrup: false
  },
  created_at: "2024-12-27T14:30:00Z"
}
```

### 6.2 데이터 스키마를 위한 사용자 흐름

#### 6.2.1 메뉴 조회 및 표시
1. 사용자가 '주문하기' 화면에 접속
2. 프런트엔드는 `GET /api/menus` API를 호출하여 메뉴 목록을 요청
3. 백엔드는 데이터베이스의 `Menus` 테이블에서 모든 메뉴 정보를 조회
4. 조회된 메뉴 정보를 프런트엔드로 반환 (이름, 설명, 가격, 이미지)
5. 프런트엔드는 받은 메뉴 정보를 화면에 카드 형태로 표시
6. 관리자 화면에서는 `GET /api/menus` API를 호출하여 재고 수량(`stock`) 정보도 함께 조회하여 표시

#### 6.2.2 장바구니 관리 (프런트엔드 내부 처리)
1. 사용자가 메뉴 카드에서 옵션을 선택하고 "담기" 버튼 클릭
2. 선택한 메뉴와 옵션 정보는 프런트엔드의 장바구니 상태에 저장 (서버 요청 없음)
3. 장바구니에 담긴 아이템 목록과 총 금액을 화면에 실시간으로 표시

#### 6.2.3 주문 생성 및 저장
1. 사용자가 장바구니에서 "주문하기" 버튼 클릭
2. 프런트엔드는 `POST /api/orders` API를 호출하여 주문 정보를 전송
3. 전송 데이터 구조:
   ```javascript
   {
     items: [
       {
         menuId: 1,
         menuName: "아메리카노(ICE)",
         quantity: 2,
         options: {
           addShot: true,
           addSyrup: false
         },
         price: 4500
       }
     ],
     totalAmount: 12500
   }
   ```
4. 백엔드는 다음 작업을 수행:
   - `Orders` 테이블에 주문 정보 저장 (주문 일시, 총 금액, 상태: 'received')
   - 각 주문 아이템을 `OrderItems` 테이블에 저장
   - 주문된 메뉴의 재고를 차감 (`Menus` 테이블의 `stock` 필드 업데이트)
   - 재고 부족 시 에러 응답 반환
5. 주문 생성 성공 시 주문 ID와 함께 성공 응답 반환
6. 프런트엔드는 성공 메시지를 표시하고 장바구니를 비움

#### 6.2.4 주문 현황 조회 및 상태 관리
1. 관리자가 관리자 화면의 '주문 현황' 영역에 접속
2. 프런트엔드는 `GET /api/orders` API를 호출하여 주문 목록을 요청
3. 백엔드는 `Orders` 테이블과 `OrderItems` 테이블을 JOIN하여 주문 정보를 조회
4. 주문은 최신순으로 정렬하여 반환 (주문 일시 기준 내림차순)
5. 프런트엔드는 받은 주문 정보를 화면에 표시
6. 관리자가 주문 상태 변경 버튼 클릭:
   - "제조 시작" 클릭 시: `PATCH /api/orders/:orderId/status` API 호출, 상태를 'in_progress'로 변경
   - "제조 완료" 클릭 시: `PATCH /api/orders/:orderId/status` API 호출, 상태를 'completed'로 변경
7. 백엔드는 `Orders` 테이블의 해당 주문의 `status` 필드를 업데이트
8. 프런트엔드는 업데이트된 주문 목록을 다시 조회하여 화면에 반영

### 6.3 API 설계

#### 6.3.1 메뉴 관련 API

**GET /api/menus**
- **설명**: 커피 메뉴 목록을 조회합니다.
- **요청**: 없음
- **응답**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": 1,
        "name": "아메리카노(ICE)",
        "description": "시원한 아이스 아메리카노",
        "price": 4000,
        "imageUrl": "https://images.unsplash.com/...",
        "stock": 10
      }
    ]
  }
  ```
- **에러 응답**:
  ```json
  {
    "success": false,
    "error": "메뉴 조회 중 오류가 발생했습니다."
  }
  ```

**GET /api/menus/:menuId**
- **설명**: 특정 메뉴의 상세 정보를 조회합니다.
- **요청 파라미터**: `menuId` (경로 파라미터)
- **응답**:
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "name": "아메리카노(ICE)",
      "description": "시원한 아이스 아메리카노",
      "price": 4000,
      "imageUrl": "https://images.unsplash.com/...",
      "stock": 10
    }
  }
  ```

**PATCH /api/menus/:menuId/stock**
- **설명**: 특정 메뉴의 재고를 수정합니다.
- **요청 파라미터**: `menuId` (경로 파라미터)
- **요청 본문**:
  ```json
  {
    "stock": 15
  }
  ```
- **응답**:
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "stock": 15
    }
  }
  ```

#### 6.3.2 주문 관련 API

**POST /api/orders**
- **설명**: 새로운 주문을 생성합니다.
- **요청 본문**:
  ```json
  {
    "items": [
      {
        "menuId": 1,
        "menuName": "아메리카노(ICE)",
        "quantity": 2,
        "options": {
          "addShot": true,
          "addSyrup": false
        },
        "price": 4500
      }
    ],

  }
  ```
- **응답**:
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "orderDate": "2024-12-27T14:30:00Z",
      "totalAmount": 12500,
      "status": "received"
    }
  }
  ```
- **에러 응답 (재고 부족)**:
  ```json
  {
    "success": false,
    "error": "재고가 부족합니다.",
    "details": {
      "menuId": 1,
      "menuName": "아메리카노(ICE)",
      "availableStock": 1,
      "requestedQuantity": 2
    }
  }
  ```

**GET /api/orders**
- **설명**: 주문 목록을 조회합니다. 관리자 화면의 '주문 현황'에 사용됩니다.
- **쿼리 파라미터** (선택):
  - `status`: 주문 상태 필터 ('received', 'in_progress', 'completed')
  - `limit`: 조회할 최대 개수 (기본값: 50)
- **응답**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": 1,
        "orderDate": "2024-12-27T14:30:00Z",
        "totalAmount": 12500,
        "status": "received",
        "items": [
          {
            "menuId": 1,
            "menuName": "아메리카노(ICE)",
            "quantity": 2,
            "options": {
              "addShot": true,
              "addSyrup": false
            },
            "price": 4500
          }
        ]
      }
    ]
  }
  ```

**GET /api/orders/:orderId**
- **설명**: 특정 주문의 상세 정보를 조회합니다.
- **요청 파라미터**: `orderId` (경로 파라미터)
- **응답**:
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "orderDate": "2024-12-27T14:30:00Z",
      "totalAmount": 12500,
      "status": "received",
      "items": [
        {
          "menuId": 1,
          "menuName": "아메리카노(ICE)",
          "quantity": 2,
          "options": {
            "addShot": true,
            "addSyrup": false
          },
          "price": 4500
        }
      ]
    }
  }
  ```

**PATCH /api/orders/:orderId/status**
- **설명**: 주문 상태를 변경합니다.
- **요청 파라미터**: `orderId` (경로 파라미터)
- **요청 본문**:
  ```json
  {
    "status": "in_progress"
  }
  ```
- **응답**:
  ```json
  {
    "success": true,
    "data": {
      "id": 1,
      "status": "in_progress"
    }
  }
  ```
- **에러 응답 (잘못된 상태 값)**:
  ```json
  {
    "success": false,
    "error": "유효하지 않은 주문 상태입니다."
  }
  ```

#### 6.3.3 옵션 관련 API (선택사항)

**GET /api/options**
- **설명**: 사용 가능한 옵션 목록을 조회합니다.
- **쿼리 파라미터** (선택):
  - `menuId`: 특정 메뉴에 적용 가능한 옵션만 조회
- **응답**:
  ```json
  {
    "success": true,
    "data": [
      {
        "id": 1,

        "price": 500,
        "menuId": null
      },
      {
        "id": 2,
        "name": "시럽 추가",
        "price": 0,
        "menuId": null
      }
    ]
  }
  ```

### 6.4 에러 처리

모든 API는 다음 형식의 에러 응답을 반환합니다:

```json
{
  "success": false,
  "error": "에러 메시지",
  "details": {}  // 선택사항: 상세 에러 정보
}
```

**HTTP 상태 코드:**
- `200`: 성공
- `400`: 잘못된 요청 (유효성 검사 실패 등)
- `404`: 리소스를 찾을 수 없음
- `500`: 서버 내부 오류

### 6.5 데이터베이스 관계

- `Menus` (1) : (N) `OrderItems`: 하나의 메뉴는 여러 주문 아이템에 포함될 수 있음
- `Orders` (1) : (N) `OrderItems`: 하나의 주문은 여러 주문 아이템을 포함할 수 있음
- `Menus` (1) : (N) `Options`: 하나의 메뉴는 여러 옵션을 가질 수 있음 (menu_id가 NULL이면 모든 메뉴에 적용)

### 6.6 구현 우선순위

1. **1단계 (필수)**: 메뉴 조회 API, 주문 생성 API, 주문 목록 조회 API
2. **2단계 (필수)**: 주문 상태 변경 API, 재고 수정 API
3. **3단계 (선택)**: 옵션 관련 API, 특정 주문 조회 API
